## Layman Summary

A benchmark dataset testing whether AI models can correctly identify whether they used facts from a document or general world knowledge when answering questions, built from four curated sources.

## Full Summary

This artifact prepares a 1,995-row multi-regime dataset for the Provenance-Polarity Gate experiment, standardized to the exp_sel_data_out JSON schema. Four datasets were selected and processed:

1. **CLUTRR/v1** (495 examples, `universal_rule` regime): Kinship-chain stories where entity facts are document-extensional and kinship axioms are world-knowledge-universal. Includes gold proof chains (`metadata_gold_premises`), hop depth (1–8), and relation type (`metadata_f_comb`). Sourced from `kendrivp/CLUTRR_v1_extracted` (HuggingFace mirror of EMNLP 2019 benchmark). Stratified sample across task_name hop-depth buckets.

2. **hitachi-nlp/proofwriter_processed_OWA** (500 examples, `universal_rule` regime): Logical inference over theory+question pairs where rules are explicitly stated inside the document, making them document-stipulated bridging premises. Pre-split by OWA depth 0–3 (125 per depth). Includes gold proof trees (`metadata_proof_tree`). Best dataset for testing whether the model identifies rules from the document versus world knowledge.

3. **nguyen-brat/entailment_bank** (500 examples, `universal_rule` regime): Science QA requiring world-knowledge bridging rules with no context document in the input. Gold chain-of-thought proofs (`metadata_proof`) provide step-by-step entailment chains. Tests pure world-knowledge rule retrieval.

4. **lighteval/synthetic_reasoning_natural (hard)** (500 examples, `document_stipulated` regime): Synthetic if-then rule chains embedded directly in the input text. Rules are unambiguously document-stipulated. Clean, single-hop, no noise — ideal for the baseline/easy tier of the polarity gate.

All rows validated against the exp_sel_data_out schema. Output: `full_data_out.json` (1.9 MB), `mini_data_out.json` (12 examples), `preview_data_out.json` (12 examples, strings truncated to 200 chars). All files well under 100 MB. Reproducible via `data.py` with seed=42.
