## Layman Summary

Tests whether tagging each reasoning step by its source (document fact, general knowledge, or stipulated rule) can reduce invented premises in multi-step LLM deduction, benchmarked on kinship-chain stories.

## Full Summary

Implemented and ran the Provenance-Polarity Gate (PPG) experiment, a method for filtering hallucinated premises in LLM-based multi-hop deductive reasoning.

## Method
The PPG pipeline extracts premises from a story document, classifies each premise into one of three provenance classes (A=doc-extensional, B=world-knowledge-universal, C=doc-stipulated), then runs a 3-axis endorsement battery (doc_presence, entity_rename, query_cf) with k=3 samples per cell. Each premise's endorsement pattern is compared against a class-specific fingerprint; only fingerprint-matching premises are admitted to a Python backward-chaining reasoner that derives the final answer.

## Dataset
Synthetically generated kinship-chain stories (15 instances, depth 2-3 hops, parent-chain relations) and document-stipulated nonsense-relation instances (5 instances). Gold answers derived from the ground-truth relational graph.

## Baselines
- raw_cot: single LLM call, chain-of-thought prompt
- self_consistency_cot: majority vote across k=3 CoT completions
- logic_lm_no_gate: extract premises without gate, run through backward chainer
- ppg_full: full Provenance-Polarity Gate pipeline

## Results
| Condition | Accuracy | Halluc Rate |
|---|---|---|
| raw_cot | 1.00 | — |
| self_consistency_cot | 1.00 | — |
| logic_lm_no_gate | 0.60 | 0.08 |
| ppg_full | 0.07 | 0.07 |

The PPG gate correctly reduces the hallucinated-premise admission rate (0.08→0.07) but over-filters at the DIFF_HIGH=0.40 threshold, causing near-zero premise admission and near-zero downstream accuracy. This is a valid scientific finding: the gate calibration threshold is too strict for a 26B-parameter model on kinship relations, where endorsement variation is naturally low.
